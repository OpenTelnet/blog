上次讲述了hexo的本地搭建过程，这一次来讲述如何用hexo生成真正的大家可以访问的博客吧。

更换主题

默认的主题是landscape，可以看到很简洁，如果你只是想自己写文章而不在乎谁看到，那么恭喜你够用了。不过大部分时候，我们并不满意它，也并不是那么好看。我们可以看到官网拥有207个主题(截止到2018/5/23)。这么多主题会让你挑很久，然而很遗憾的告诉你，并不是每个主题都能直接使用的。具体的你需要去该主题的github页面看别人的讨论，而且有些主题是远古时代的产物，作者可能放弃更新了。更换主题是一件十分容易的事情，仅需要下面几步。
下载主题，我就用我的主题来举一个例子，在你博客的目录下git项目到你的themes文件夹下


接下来编辑_config.yml，将其中的“theme: landscape”换成“theme: material”。这是hexo更换主题的通用步骤，不过我们的material主题作者有一些谨慎，我们下载的themes/material下有一个文件名为_config.template.yml，我们需要把它名字改成_config.yml，记住，这是hexo的配置文件，名字都是这样的，抓住事情的本质一切就会豁然开朗。接下来终端执行“hexo s”即可看到我们的主题更换成功了。

Github Page

Github是我们知道著名的代码托管网站，然而本教程并不需要你去写代码。之所以用到它是因为Github Page可以支持预览静态网页，你想想，你做个人博客不就是为了给别人看吗？只要你想要的界面出来了，你还用得着买服务器吗？

创建你的仓库

第一步，注册一个github账号，这个还用我教的话，面壁去。
然后新建一个仓库，点击右上角+，点击New repository。
Repository name里写上格式如“你的用户名.github.io”，像我的仓库名就叫做opentelnet.github.io
接下来创建仓库，点击绿色的Create repository。

生成静态网页hexo g

hexo g用来生产静态网页，保存在public文件夹下。

####上传到网页到仓库里hexo d
前文说过，你可以在github网页上将文件上传到仓库里，会比较麻烦。如果你会设置了git这是十分方便的。
在博客根目录下右键空白处选择Git Bash here，接下来执行下列操作


输入完上一条命令后会生成一条rsa密码，如我的就生成在C:\Users\用户名.ssh\id_rsa.pub文件里，用sublime之类的软件打开它，全部复制，然后打开github.com，点击settting，找到SSH and GPG keys一项，点击New SSH keys，TiTle随便输入，Key就黏贴我们刚刚输入的部分。
在终端输入命令hexo d，网页更新完成。输入你的仓库名，如opentelnet.github.io即可访问。

域名解析

域名管理页面

添加一个CNAME记录，记录值为www，域名是自己购买的域名。

github设置

打开你的仓库页面，点击仓库的setting，找到新页面里有一项github page，找到custom domain，填写你的域名，如www.opentelnet.com ，点击save保存。

Gitment

Gitment是基于仓库存储文章的评论，实际上与数据库无关，而是存在你的仓库里。

一般情况下，你的主题像我的就是themes/material下的_config.yml文件里有配置文件，ctrl+f找到comment，修改其中的几行，我把我的设置给你们看。


use后要加gitment，gitment_repo写上自己的仓库名称，gitment_owner就是自己的github用户名，gitment_client_id和gitment_client_secret需要自己去获取。
右上角点自己头像展开，点击setting，在弹出的网页中选中侧边的developer setting，接下来是OAuth Apps，点击右边的New OAuth App。
Application name：随便输入
Hompage URL:你的域名格式如https://www.opentelnet.com
Application description:随便输入
Authorization callback URL:你的域名格式如https://www.opentelnet.com
接下来点击绿色的Register application。好了，获取到了你的client id和client secret，把它们更改到配置文件里。

  • 顺带一提，你创建文件的时候就是hexo new 文章名，这个文章名千万别使用中文，这个名字是地址栏会显示的名字，而且最好别超过50个字符，中文名字你可以打开md文件给title起中文名那是没关系的。

    We talked about local establishment of hexo on local computer. Let’s make a blog website ready for everyone to visit now!

    Change a theme

    Hexo use the theme ‘Landscape’ as default. If you only write pages regardless of who can see it, it’s just ok. However, we don’t feel satisfy with it because it’s not good for us. We can see 207 themes(by 2018/5/23) onofficial website. There’s so many themes that we could cost much time to choose which one to get. Aha, I will tell you that some of them isn’t on function. You should see the discuss on their github even a few themes stop to update. It’s really simple for us to change a theme just follow my steps.
    Firstly, download your favourite theme and clone it to the floder named themes. I will take my example that I choose the theme material. Edit _config.yml:[theme: landscape] –> [theme:material]
    That’s a common step when you want to change your theme. However, the theme material is special. We should modify themes/material/_config.template.yml as themes/material/_config.yml.
    Execute [hexo s] and you can visit http://127.0.0.1:4000 to look it.

    Github Page

    As we known, github is a code hosting site but we do not need to code. We use it just for store out website for free. OMG, we can save lots of money per year.

    Create your own repo

    Firstly, please register a github account.
    Then create a repo. Click the [+] on the top and click [New repository].
    Repository name: e.g.’opentelnet.github.io’.
    Click the green button [Create repository].

    generate static pages-hexo g

    You can execute [hexo g] to generate static pages and they saved in a floder named public.

    Upload your website-hexo d

    Let’s talk about git next.
    Right click [Git Bash here] on the blank in the floder which your blog here.
    Follow the next steps: We will get a rsa password in the file [C:\Users\username.ssh\id_rsa.pub]
    Open and copy full of it.
    Visit github.com.
    Click setting and look for [SSH and GPG keys], then new it.
    TiTle: with your pleasure.
    Key： paste what we copied just now
    Execute [hexo d] to upload your website to your github repo.
    Visit your blog by typing your repo name such as [opentelnet.github.io].

    DNS

    Domain Name Service Provider

    Github Setting

    Click the save button.

    Gitment

    Gitment is a comment system which stores comments in your repo.

    Open themes/material/_config.yml.
    Ctrl + f to find [comment].
    Modify the following rows: Add [gitment] after [use].
    Add your Respo Name after [gitment_repo].
    Add your github username after [gitment_owner].
    You need to get your own gitment client id and secret by the following steps.
    Click your headings and then click setting.
    Select [developer setting] on the side bar.
    Then, click the [OAuth Appas] and new it. Here’s the example:
    Application name： with your pleasure
    Hompage URL: e.g.https://www.opentelnet.com
    Application description: with your pleasure
    Authorization callback URL: e.g.https://www.opentelnet.com
    Next, click the green button [Register application]. Okay, just put your client id and client secret to _config.yml
