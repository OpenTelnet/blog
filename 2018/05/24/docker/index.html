docker镜像与容器的区别

看这篇文章的人，应该都是刚接触docker技术的人。我姑且就举大家习以为常的例子，可能并不是那么准确。docker镜像就是一个个硬盘，里面装着不同的操作系统，而容器就是电脑，有各种硬件，就少一块硬盘，当你选择一个镜像放入容器的时候，就好像这块硬盘给了这台电脑，然后去运行它。其实更有点像网吧里的游戏被每个电脑运行这样的原理。因为docker镜像可以同时被多个容器使用。

    Difference between docker image and container

    May you all are newcomers to docker technology. Let me just give you an example of what you are used to. It may not be so accurate. A docker image is a hard disk containing different operating systems. The container is a computer. There are various kinds of hardware. There is a hard disk. When you select an image and put it into a container, it looks like the hard disk has given this computer. Then go to run it. Actually, it’s more like the principle that a game in a cyber cafe is run on every computer. Because docker images can be used by multiple containers at the same time.
